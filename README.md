# PVZ Manager

**Клиент-серверная система для автоматизации учёта деятельности пункта выдачи заказов (ПВЗ)**

---

## 📱 О проекте

PVZ Manager – это полнофункциональное решение для владельцев пунктов выдачи заказов, позволяющее вести учёт смен сотрудников, доходов от маркетплейсов (Яндекс, Wildberries), расходов ПВЗ и получать наглядную аналитику. Система состоит из веб-панели администратора на Flask и нативного iOS-приложения с поддержкой офлайн-режима.

### Основные возможности

- **Управление сотрудниками и сменами** – добавление сотрудников, фиксация смен с датой, временем, часами, ставкой, бонусами/вычетами, отметкой присутствия.
- **Учёт доходов** – импорт Excel-отчётов маркетплейсов (Яндекс, WB), календарь доходов с фильтрацией по источнику, детализация по услугам.
- **Учёт расходов** – добавление расходов по категориям (интернет, электричество, вода, охрана и др.), круговая диаграмма распределения, таблица с фильтром по месяцам.
- **Аналитический дашборд** – KPI (часы, доход, расходы, чистая прибыль), годовые графики (часы, доход/расходы), топ-5 сотрудников по часам, топ-5 услуг по доходу.
- **Офлайн-режим** (iOS) – кэширование данных в Core Data, синхронизация при восстановлении интернета.
- **Аутентификация** – сессионная (Flask‑Login) для веб и мобильного клиента, безопасное хранение учётных данных в Keychain на iOS.

## 🚀 Установка и запуск

### Требования
- Python 3.12+
- pip
- SQLite (встроенная)
- iOS 16.0+ (для мобильного приложения) или веб-браузер

### 1. Серверная часть

```bash
# Клонирование репозитория
git clone https://github.com/your-username/pvz-manager.git
cd pvz-manager

# Создание виртуального окружения
python -m venv .venv
source .venv/bin/activate   # Linux/macOS
# .venv\Scripts\activate   # Windows

# Установка зависимостей
pip install -r requirements.txt

# Запуск Flask-приложения
flask run --host=0.0.0.0 --port=5000
```

После запуска веб-панель доступна по адресу `http://localhost:5000`.  
Логин по умолчанию: `admin` / `12345`.

### 2. iOS-приложение

1. Открыть проект `PVZmanager.xcodeproj` в Xcode.
2. В файле `NetworkManager.swift` изменить `baseURL` на IP-адрес компьютера с сервером.
3. Выбрать целевое устройство (симулятор или iPhone) и запустить сборку (⌘R).

> **Важно:** на iPhone в настройках `Info.plist` уже добавлено разрешение на незащищённые HTTP-соединения. iPhone и компьютер должны быть в одной Wi-Fi сети.

## 📖 Руководство пользователя

### Веб-панель
- **Главная страница (`/dashboard`)** – выбор месяца, просмотр KPI, графиков, последних расходов.
- **Сотрудники (`/employees`)** – добавление/удаление сотрудников, просмотр их смен (модальное окно).
- **Смены (`/`)** – загрузка Excel-файла со сменами, таблица всех смен.
- **Доходы (`/income_calendar`)** – календарь доходов, переключение источника (Яндекс / WB / все), детализация по дням. Загрузка отчётов через отдельные страницы `/upload_income` и `/upload_wb`.
- **Расходы (`/expenses`)** – добавление расхода, таблица, круговая диаграмма.

### iOS-приложение
- Аналогичный функционал, оптимизированный для мобильного использования. Поддерживает офлайн-режим и биометрическую защиту входа (опционально).

## 🔧 API эндпоинты (основные)

| Метод | Эндпоинт                       | Описание                          |
|-------|--------------------------------|-----------------------------------|
| POST  | `/api/login`                   | Вход (установка сессии)           |
| POST  | `/api/logout`                  | Выход                             |
| GET   | `/api/employees`               | Список сотрудников                |
| GET   | `/api/shifts`                  | Смены (с фильтром по месяцу)      |
| GET   | `/api/income`                  | Доходы за месяц (с параметром source) |
| GET   | `/api/expenses`                | Расходы за месяц                  |
| GET   | `/api/dashboard`               | Агрегированные KPI и графики      |

Подробная спецификация API приведена в Приложении А курсовой работы.
